### 6.2 稳定时间计算口径

当前 UI 中稳定时间不再取 `AST.INPOS` 与 `MST.INPOS` 的差值，也不再直接使用旧版误差窗口含义，而是：

1. 从 `SETTLE_POS_DATA` 中识别“开始运动”的采样点
2. 从该采样点之后寻找 `MST.INPOS` 首次置位
3. 用两者的采样点差乘以 `sampleIntervalMs` 得到单次稳定时间
4. 报告指标只看 `Max Settling Time`
5. 再与配置中的 `max_time` 比较，给出是否满足结论

### 6.3 稳定时间 buffer 步进逻辑

稳定时间脚本目前已经改成真正按步长扫描：

- 新增控制参数 `SETTLE_STEP_DISTANCE`
- 默认从负限位开始
- 以 `SETTLE_STEP_DISTANCE` 逐步向正限位移动并逐点测试
- 到达靠近正限位后，将正限位作为最后一个正向点
- 然后按相同点位反向逐步回到负限位并继续测试
- 如果没有显式设置 `SETTLE_STEP_DISTANCE`，才回退到按 `SETTLE_POINT_COUNT` 等分

同时已经做过容量扩展，避免“小步长 + 大行程”一上来就因为点位太多报错：

- `SETTLE_POINT_POS`：`20 -> 100`
- `SETTLE_MOVE_TARGET / SETTLE_POS_DATA / SETTLE_MOVE_DIRECTION / SETTLE_AST_INPOS / SETTLE_MST_INPOS`：`40 -> 200`
- UI 侧稳定时间回读 `moveCount` 上限同步扩到 `200`

### 6.4 速度均匀性与动态能力

- 速度均匀性不再只看单次曲线，而是按 `speed_min ~ speed_max` 的多个测试点生成结果
- 每个速度点的命令参数当前约定为：
  - `ACC = SPEED * 10`
  - `JERK = ACC * 10`
- 控制器侧在检测到 `GPHASE = 4` 后才开始 `DC`
- 如果某个速度点正向或反向未进入匀速段，该速度点跳过
- 页面会显示各测试点的速度均匀性连线
- 动态能力曲线会标出最大速度点
